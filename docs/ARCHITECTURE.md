# 架构说明

## 1. 总览

本项目采用“共享领域核心 + 薄平台壳”结构。Windows 和 Android 复用扫描、证据验证、上下文压缩与持久化语义；平台层只处理文件授权、安全存储、长任务生命周期和窗口/通知等原生能力。

```text
apps/client  React + TypeScript 响应式界面
        │  typed commands / events
        ▼
apps/desktop  Tauri 2 应用壳
        │
        ├── Windows：文件选择、Credential Manager、窗口与安装包
        ├── Android：SAF URI、Keystore 封装、前台任务/通知
        │
        ▼
Rust command facade
        ├── novel-import：识别、解码、分章、SourceLocator
        ├── novel-core：任务、扫描、证据、上下文检查点
        ├── rulepack：版本化社区规则与用户覆盖
        ├── provider adapters：BYOK 模型调用与结构校验
        └── SQLite repository：本地任务状态和可恢复检查点
```

首发不包含 iOS target。核心不得依赖只能在 Windows 上运行的子进程；Android 也不依赖 Python sidecar。

## 2. 模块职责与依赖方向

| 模块 | 负责 | 不负责 |
| --- | --- | --- |
| `apps/client` | 人性化对话式工作区、导入/规则/进度/证据界面、无障碍与响应式布局 | 直接读文件、保存明文密钥、相信模型摘录 |
| `apps/desktop` | Tauri 命令和事件、平台能力桥接、应用生命周期 | 复制一套扫描业务规则 |
| `crates/novel-import` | 格式探测、编码、正文与章节、来源锚点、能力报告 | 调用模型、判断雷点 |
| `crates/novel-core` | 领域模型、扫描编排、候选验证、上下文快照、确定性测试提供器 | 持有平台密钥、实现具体 UI |
| `packages/rulepack` | Schema、版本化规则、来源、核验状态、用户选择示例 | 把社区清单包装成客观事实 |
| provider adapters | 请求映射、鉴权、超时/重试、结构化响应与用量归一 | 绕过核心证据验证、将模型输出直接写成确认命中 |
| SQLite repository | 文档元数据、任务、规则选择、检查点、命中和用量的事务性保存 | 存储 API Key 或默认上传完整原书 |

依赖方向必须从平台/适配器指向领域契约。`novel-core` 不导入 React、Tauri、Android 或具体供应商 SDK。

## 3. 多模型 BYOK

提供器通过统一 `ModelProvider` 契约接入，配置分为两类：

- 可持久化的非秘密配置：提供器 ID、模型 ID、兼容 API 地址、温度/预算、超时策略。
- 秘密：API Key 或令牌。Windows 使用系统凭据设施；Android 使用 Keystore 中的密钥封装加密数据。前端只得到“已配置/未配置”和脱敏标识。

首版优先实现 OpenAI-compatible 请求适配，以覆盖明确兼容该协议的服务；对 Anthropic、Gemini 等协议差异明显的服务使用原生适配器。不能仅更换 URL 就声称兼容。

所有提供器必须经过同一出站边界：

1. 显示将要使用的提供器以及是否会把正文发送到外部服务。
2. 仅发送当前章节和压缩后的必要上下文，不发送本地路径、密钥或无关书籍。
3. 设置连接、首字节和总请求超时，按错误类型决定是否重试；重试必须幂等并记录用量边界。
4. 对结构化响应做字段、规则 ID、置信度和 UTF-8 范围校验。
5. 模型没有文件、Shell、数据库或任意工具权限；小说正文始终按不可信数据处理。

`DeterministicTestProvider` 仅供离线测试、UI 演示和回归测试。它不需要密钥，也不能在产品界面中伪装为真实智能扫描。

## 4. 长上下文压缩与断点恢复

全书不会一次性塞进模型上下文。逐章流水线维护结构化 `ContextSnapshot`：

- 已处理章节 ID 与文档/章节内容指纹；
- 有长度预算的滚动摘要；
- 人物、别名、关系和关键状态；
- 跨章事件/承诺及仍待后文确认的候选；
- 已确认命中的引用 ID，而不是重复堆叠长摘录。

处理第 N 章时只组合：选中规则、当前章节、N-1 的快照和必要的未决候选。完成后生成 N 的新快照，并在同一 SQLite 事务中保存章节进度、命中、用量和检查点。

压缩文本是推理记忆，不是证据。摘要中出现的内容不得直接生成“已确认”命中。恢复扫描前必须比较文档和已处理章节指纹；不匹配时使旧检查点失效或从变化点重新扫描，不能把旧证据挂到新文本。

## 5. 证据与章节定位

导入层为每章提供格式相关 `SourceLocator`：

| 格式 | 目标定位信息 |
| --- | --- |
| TXT / Markdown | 章节、原文行号、解码后 UTF-8 字节范围 |
| EPUB | spine 顺序、资源 href、fragment/CFI 或段落 |
| PDF | 页码、文本块；可用时追加坐标范围 |
| DOCX | 标题层级、段落起止序号 |
| HTML | 资源、稳定 DOM 路径/选择器和文本范围 |
| MOBI / AZW3 | 内容资源和电子书 location，仅限无 DRM 文档 |
| ZIP / 7Z | 归档条目路径 + 内部文档自己的定位器 |

模型只允许返回当前章节中的 UTF-8 字节范围和理由。核心检查边界、字符边界、规则 ID、章节指纹后，从原文切片重建 `exact_quote` 并计算引用指纹。

- `suspected`：模型提出了候选，但证据缺失或暂时无法从原文合法重建；仅作为待复核候选，不能使用“实锤”等确认措辞。
- `pending_confirmation`：已有可从原文重建的精确线索，但仍需要后续章节、人物关系事实或用户判断才能定性。
- `confirmed`：所有证据范围都能从原文重建，章节/内容指纹一致，而且不再依赖后续确认。
- `rejected`：候选已被后文或人工复核排除；保留状态用于持久化与审计，但不作为有效命中展示。

扫描器可以直接产出前三种状态；`rejected` 由后续语境或人工复核写入。任何状态都不得编造章节或摘录，疑似与待确认候选在上下文压缩中保持为未决项。

## 6. 社区规则包

yy小说吧规则包必须携带 `pack id + version + schema version`。每条规则至少包含稳定 ID、社区名称、解释、判据、排除条件、待确认条件、默认分类/等级、用户可覆盖项、来源引用和核验状态。

核验遵循：

1. 原帖和后续社区用法分开记录，保留访问日期与可追踪链接。
2. 只有逐项核对过名称和含义的条目才标为 `verified`；未核验或占位条目默认关闭。
3. 帖子数量（例如种子包的 11 个雷点、21 个郁闷点）只用于发现遗漏，不能代替语义核验。
4. 同一概念在两个分类中出现时使用不同规则 ID，用户可独立开关和覆盖等级，不能在导入时去重丢失。
5. 规则升级不悄悄改变旧任务结果；任务记录所用包版本，重扫时由用户选择迁移。
6. 用户自定义规则与社区包分层保存，更新官方/社区包不能覆盖个人选择。

## 7. SQLite 数据与事务边界

建议的逻辑表如下，最终迁移文件以实现为准：

| 数据域 | 关键内容 |
| --- | --- |
| `documents` / `chapters` | 显示信息、格式、来源授权引用、顺序、内容指纹、位置元数据 |
| `scan_tasks` / `rule_selections` | 状态、预算、规则包版本、规则开关和等级覆盖 |
| `scan_checkpoints` | 最后完成章节、上下文快照、文档指纹、版本 |
| `findings` / `evidence_anchors` | 规则、状态、理由、原文锚点、引用/章节指纹、提供器标记 |
| `usage_events` | （计划 S3）提供器归一后的输入/输出单位、请求状态和时间，不含正文 |
| `provider_profiles` | 非秘密配置和安全存储中的 secret reference；不含明文 Key |

SQLite 使用迁移版本管理，开启外键并采用短事务。完成一章时原子提交检查点和结果；取消网络请求不应留下“已完成但无结果”的章节。删除书籍时同时清理派生正文、命中和检查点，并单独处理平台持久 URI 授权。

数据库不应被描述成已加密，除非实际启用了并测试了数据库加密。首发安全基线是应用沙箱、最小化存储、系统秘密存储和可见的本地删除；更高敏感场景可另行评估数据库加密。

## 8. 安全与隐私边界

- 小说文件、规则包文本和模型响应均是不可信输入；防止路径穿越、压缩炸弹、超大文件、恶意 HTML/XML 和提示注入。
- ZIP/7Z 解包设置文件数、单文件大小、总解压大小和嵌套层数上限，拒绝绝对路径及逃逸目标目录的条目。
- EPUB/DOCX/XML 解析禁用外部实体和网络取回；HTML 导入不执行脚本。
- 不尝试绕过 MOBI/AZW3/PDF 的 DRM 或密码保护。
- 日志默认不写正文、摘录、密钥、完整本地路径或 SAF URI；诊断导出须先脱敏并由用户确认。
- 模型调用前展示数据出站事实；本地导入和历史浏览不要求联网。
- 内容指纹用于检查点一致性，不自动等同于密码学防篡改。发布文件校验应使用专门的密码学哈希。
- Android 长任务使用受平台约束的前台任务/持久工作机制并持续落盘；进程被系统回收后从最后完整事务恢复。
