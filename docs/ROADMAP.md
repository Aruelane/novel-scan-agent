# 阶段路线与验收清单

本文把项目计划拆成可以在 issue、里程碑和 PR 中逐项勾选的门禁。任何阶段都不得用演示数据代替真实能力声明。

## S1：跨端基础骨架

- [ ] React/TypeScript 前端测试和生产构建在 Node 24 通过。
- [ ] 响应式界面覆盖桌面三栏和 Android 单栏/底部导航，不以鼠标悬停作为唯一操作。
- [ ] Tauri 2 壳位于 `apps/desktop`，原生 Rust 目录位于 `apps/desktop/src-tauri`，前端继续位于 `apps/client`。
- [ ] Rust workspace 在 Windows runner 完成 `fmt --check`、`check --workspace --all-targets` 和 `test --workspace --all-targets`。
- [ ] `novel-core`、`novel-import`、模型提供器和规则包边界可独立测试。
- [ ] 规则包校验检查版本、唯一 ID、引用、核验状态和用户覆盖。
- [ ] README 不把测试提供器称作真实模型，也不要求启动时配置 API Key。

退出条件：所有 CI job（frontend, rulepack, migration, rust）全绿，文档中的”当前支持”与代码能力注册表一致。Windows 壳启动冒烟测试未执行，留待 S6。

## S2：多格式导入与来源定位

- [ ] TXT：UTF-8/BOM、UTF-16、GBK/GB18030；换行和章节标题夹具齐全。
- [ ] Markdown：标题分章和原始行号可复核。
- [ ] EPUB：spine/目录顺序、资源路径和片段/段落定位。
- [ ] DOCX：标题层级、段落定位；页眉页脚处理规则明确。
- [ ] HTML：正文提取不执行脚本，保留资源与 DOM/文本定位。
- [ ] PDF：先支持有文本层文件；逐页记录提取失败和页码。
- [ ] MOBI/AZW3、ZIP/7Z 保持“受限/待实现”，直到有安全解析与回证测试。
- [ ] 旧 DOC 和扫描版 PDF 显示转换/OCR 提示，不静默产出空书。
- [ ] Windows 路径和 Android SAF URI 均通过同一导入契约进入核心。

退出条件：所有标为“可解析”的格式都具备正常、损坏、超大和定位回归测试。

## S3：全书扫描与上下文

- [ ] 按章扫描，固定上下文预算，不因书长线性扩大单次请求。
- [ ] 快照包含滚动摘要、实体/关系/事件状态和未决候选。
- [ ] 摘要与证据严格分离；确认命中必须重新匹配原文。
- [ ] 每章结果、用量和检查点在单一 SQLite 事务提交。
- [ ] 暂停、取消、重试、应用崩溃和系统回收后均可恢复。
- [ ] 文档或章节指纹变化会阻止复用过期检查点。
- [ ] 可见进度基于已完成章节/字符，不伪造模型百分比。

退出条件：至少三种长度档位的测试书完成全覆盖，恢复前后命中和证据一致。

## S4：BYOK 与提供器

- [ ] 提供器配置把公开字段和 secret reference 分离。
- [ ] Windows 系统凭据存储与 Android Keystore 封装均有读写/删除测试。
- [ ] 至少一个 OpenAI-compatible 适配器和一个不同协议的原生适配器可用。
- [ ] 用户可选择模型、兼容端点、预算和超时，并能测试连接。
- [ ] 请求/响应结构校验、限流退避、取消和幂等重试可观察。
- [ ] 正文出站前有清晰提示；本地功能无 Key 可用。
- [ ] 日志、错误报告和 SQLite 中没有明文 Key 或整章正文。

退出条件：两个真实提供器完成同一契约测试；密钥扫描与日志检查无泄露。

## S5：规则核验和用户定制

- [ ] 逐项核对用户提供的 yy小说吧规则帖及必要的吧内补充内容。
- [ ] 每条默认启用规则都有可追踪来源、判据、排除项和复核记录。
- [ ] 11 雷点 + 21 郁闷点数量与来源版本匹配，但数量不替代语义复核。
- [ ] “接盘”等跨分类概念保留两条可独立设置的规则。
- [ ] 用户可按书/全局预设开关规则并覆盖雷点/郁闷点等级。
- [ ] 规则包升级显示差异，不改写历史任务所用版本。
- [ ] 自定义规则可导入导出，且与社区规则来源明确区分。

退出条件：核验记录可由另一位维护者复查，默认规则不存在无来源占位项。

## S6：Windows 与 Android 产品化

- [ ] Windows 安装、升级、卸载和文件选择冒烟通过。
- [ ] Android 原生 scaffold 由 Tauri 2 CLI 生成并提交必要配置，不提交本机 SDK 路径和签名密钥。
- [ ] Android SAF 支持选择单文件并保留必要的读取授权；错误权限可恢复。
- [ ] Android 长扫描以前台通知/平台持久任务运行，支持用户停止。
- [ ] 小屏、横屏、显示缩放、软键盘和系统返回键行为通过真机验证。
- [ ] Android Keystore 封装的密钥在清除数据/删除配置时符合预期。
- [ ] GitHub 手动 Android workflow 能构建 aarch64 debug APK；发布另用受保护的签名流程。
- [ ] 仓库中没有 iOS target、证书或商店发布依赖。

退出条件：Windows 设备和至少一台 Android 真机完成“导入—选择规则—扫描—暂停恢复—定位证据”闭环。

## S7：发布门禁

- [ ] CI、格式夹具、恢复测试、规则包校验全部通过。
- [ ] 安全测试覆盖恶意归档、损坏文档、HTML/XML 外部资源和提示注入。
- [ ] 隐私说明列出各提供器的数据出站行为和本地删除路径。
- [ ] Windows 资产和 Android APK 使用发布签名，并生成密码学校验值。
- [ ] GitHub Release 清楚列出支持格式、受限格式、已知问题和迁移说明。
- [ ] 发布流程不包含用户测试书、API Key、keystore 或签名口令。

退出条件：从干净环境按文档可复现构建，发布资产通过安装与升级冒烟测试。
